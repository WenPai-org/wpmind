# 文派叶子 (wp-china-yes) 交叉销售策略

> 文档版本: 1.0 | 创建日期: 2026-02-15 | 状态: 规划中
> 关联文档: [WENPAI-GLOBAL-STRATEGY.md](./WENPAI-GLOBAL-STRATEGY.md)

## 1. 战略定位

文派叶子是文派生态中用户基础最大的产品，定位为**免费流量引擎**。

核心优势：
- **触达面广**: 中国 WordPress 用户刚需（解决 GFW/速度问题）
- **高频使用**: 每次 WordPress 更新/插件安装都触发
- **信任基础**: 解决实际痛点，用户信任度高
- **数据丰富**: Bridge 客户端 v2.1 每日遥测完整站点画像

**结论: 保持完全免费，通过交叉销售变现。**

---

## 2. 现有数据资产

Bridge 客户端 (class-site-health.php) 每日遥测收集的数据：

| 数据类型 | 字段 | 推荐价值 |
|----------|------|---------|
| 插件列表 | slug, version, active, author | 精准匹配替代/增强产品 |
| WooCommerce | wc_version, products, orders, gateways | 识别电商用户，推荐集市 |
| 平台信息 | php_version, extensions, memory | 站点健康建议（建立信任） |
| 主题信息 | slug, is_block_theme, is_child_theme | 推荐兼容主题/模板 |
| 翻译数据 | type, slug, language | 推荐翻译服务 |

## 3. 三层漏斗策略

### 第一层：被动展示（所有用户）

**触点: 叶子设置页底部 "文派生态" 区块**

不是广告，而是**生态状态面板**：
- 显示用户已安装的文派产品和版本状态
- 显示未安装但与用户场景匹配的产品（最多 2 个）
- 简洁卡片式设计，与叶子设置页风格一致

**触点: 更新日志**

叶子每次更新时，changelog 中自然提及生态新产品：
> v3.9.1 — 新增文派云桥客户端 v2.1。推荐搭配 WPMind 使用 AI 内容生成。

**实施优先级: Phase 1.5（静态推荐，无需服务端支持）**

### 第二层：场景化推荐（匹配用户）

**触点: Bridge 遥测驱动的智能推荐**

遥测上报时，服务端基于站点画像返回个性化推荐：

| 用户画像 | 推荐产品 | 推荐理由 |
|---------|---------|---------|
| 安装了 WooCommerce | 集市商户入驻 | "你在运营电商站，文派集市可以帮你分发产品" |
| 有内容发布（博客/新闻站） | WPMind Free | "AI 辅助内容创作，支持 10+ 国产 AI" |
| PHP < 8.0 或缺少关键扩展 | 站点健康建议 | 提供价值而非推销，建立信任 |
| 已安装 WPMind Free | WPMind Pro | "解锁更多 AI 服务商和 SEO 工具" |
| 开发者（多个自定义插件） | 飞码托管 + SDK | "在飞码托管你的插件，一键发布" |

**API 设计:**

```
POST /api/v1/telemetry
Request: { site_uuid, plugins, woocommerce, platform, ... }

Response: {
  "status": "ok",
  "recommendations": [
    {
      "product": "wpmind",
      "title": "文派心思 — AI 内容助手",
      "reason": "检测到你的站点有活跃的内容发布...",
      "cta_url": "https://wenpai.org/plugins/wpmind/",
      "cta_text": "免费安装",
      "priority": 1,
      "dismiss_key": "wpmind_rec_v1"
    }
  ]
}
```

客户端缓存推荐数据 7 天，用户关闭后通过 dismiss_key 记录不再显示。

**实施优先级: Phase 2（需要服务端推荐引擎）**

### 第三层：深度集成（活跃用户）

**触点: Dashboard Widget**

WordPress 后台仪表盘添加 "文派生态" 小组件：
- 显示站点健康评分
- 显示已安装文派产品状态
- 显示可用更新和推荐

**触点: WPMind 功能预览**

在叶子设置页中嵌入 WPMind 的轻量级功能预览：
- "AI 内容助手" 体验入口（调用 WPMind Free API）
- 让用户在不安装 WPMind 的情况下体验核心功能
- 体验后引导安装完整版

**实施优先级: Phase 3（需要产品成熟后）**

## 4. Admin Notice 策略（极度克制）

**只在以下场景使用一次性 admin notice（可关闭）：**
- WPMind 首次发布时（一次性公告）
- 文派集市上线时（一次性公告）
- 用户站点检测到安全风险时（提供价值）

**绝对不做：**
- 重复弹窗
- 不可关闭的通知
- 影响后台性能的推荐
- 每次更新都弹通知

---

## 5. 关键原则

1. **价值优先** — 每个推荐必须解决用户的实际问题，不是"我们有个新产品"
2. **数据驱动** — 基于遥测数据做精准匹配，不做无差别推送
3. **可关闭** — 所有推荐都可以一键关闭，关闭后不再出现
4. **不影响性能** — 推荐数据随遥测返回，不增加额外 HTTP 请求
5. **渐进式** — 先从设置页静态推荐开始，验证转化率后再做智能推荐
6. **尊重用户** — 免费产品的信任是最宝贵的资产，不能透支

## 6. 转化漏斗预估

```
叶子活跃用户 (假设 10,000)
  -> 访问设置页 (60% = 6,000)
    -> 看到推荐区 (100% = 6,000)
      -> 点击了解 (5% = 300)
        -> 安装 WPMind Free (50% = 150)
          -> 升级 Pro (10% = 15)
            -> 年收入: 15 x 199 = 2,985

加上集市产品推荐、开发者工具推荐：
整体交叉销售收入预估 10,000-30,000/年（保守）
```

数字不大，但这是**零成本获客**。随着用户基础增长，漏斗自然放大。

## 7. 实施路线图

### v1 — 静态推荐（Phase 1.5, 3-6 个月）

- [ ] 叶子设置页底部添加 "文派生态" 卡片区
- [ ] 硬编码推荐内容（WPMind、集市）
- [ ] 可关闭，关闭状态存 user_meta
- [ ] 更新日志中提及生态产品
- [ ] 埋点追踪点击率

### v2 — 智能推荐（Phase 2, 6-12 个月）

- [ ] Bridge 遥测 API 返回 recommendations 字段
- [ ] 服务端推荐引擎（基于插件列表、WC 数据等）
- [ ] 客户端渲染推荐卡片（7 天缓存）
- [ ] dismiss_key 机制
- [ ] A/B 测试不同推荐文案

### v3 — 深度集成（Phase 3, 12+ 个月）

- [ ] Dashboard Widget
- [ ] WPMind 功能预览嵌入
- [ ] 捆绑销售入口（叶子 Pro + WPMind Pro 套餐）
- [ ] 推荐效果分析面板

## 8. 度量指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 推荐区展示率 | >50% | 设置页访问用户中看到推荐的比例 |
| 推荐点击率 | >3% | 看到推荐后点击的比例 |
| 安装转化率 | >30% | 点击后实际安装的比例 |
| 付费转化率 | >5% | Free 用户升级 Pro 的比例 |
| 推荐关闭率 | <20% | 关闭推荐的比例（过高说明太打扰） |
| NPS 影响 | 无下降 | 交叉销售不应降低用户满意度 |
